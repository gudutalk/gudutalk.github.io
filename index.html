SEDA An Architecture for Well-Contained, Scalable Internet Services

3. 阶段式的事件驱动架构(The Staged Event-Driven Architecture)

在这一节，我们提出了一个使得高并发，负载调节(load conditioning)和容易操纵(ease of engineering)
的互联网服务成为可能的新的软件架构–阶段式的事件驱动架构(Staged Event-Driven Architecture–SDEA)。
SEDA通过事件队列分开把一个应用分解为由不同阶段组成的网络，并且引入了动态资源控制器，从而允许应用
根据负载的变化进行动态调整。图5给出了一个使用了SEDA架构的服务的整体设计。

3.1 目标

下面列出了SEDA的主要设计目标：

支持大规模并发:

为了避免多线程造成的性能下降，SEDA尽可能使用事件驱动的执行模型这同时也要求系统提
供高效和可伸缩的输入/输出元语

简化良态服务的构建

为了简化构建互联网服务的复杂性， SEDA将调度(scheduling)和资源管理的细节与开发者
进行隔离。同时支持模块化构建这些应用，并提供调试和性能分析的支持。

使自省成为可能

应用应该可以分析请求流，从而适应变化的负载。例如，系统应该能够对请求进行过滤和
区分优先级，以支持由于高负载退化了的服务(degraded service)。

支持自调节式的资源管理

系统应该不需要根据预先知道的应用对资源的需求以及客户端的负载特点，就可以动态地调
整它的资源管理参数，以达到目标性能。
例如，基于观察到的并发需求，自动决定分配给某个阶段的线程数，而不是由程序员或者系
统管理员设定一个固定的数值

3.1 以“阶段”作为耐用的积木式元件

SEDA的基本处理单元就是“阶段”，如图6所示，如下所述，一个“阶段”是一个自包含的应用组件，
包括事件处理程序，一个事件队列和一个线程池。每个“阶段”由一个控制器(controller)管理，
负责调度和分配线程。

  • concurrency
  • event driven
  • scalable

分红模式，股权与收益权分立，从而激励管理团队，具体如下：
假如公司盈利，税后利润分为：50%，30%，20%三个部分，50%部分按照股权结构分红，让
每个投资的员工都能感受到投资的收益，做到及时激励。总经理至少有10%的股份，因此可以
拿到5%的税后利润，30%的利润留下来作为滚动发展的资金，20%作为管理团队优先分红，也
就是管理团队的分红，20%的团队分红总经理可以拿到一半，因此总经理有10%的股权，可以
拿到15%的利润分红。这个模式的精妙之处在于，管理团队占有的是收益权，而不是公司股

安装 Python

  1. 从https://www.python.org/downloads/windows/下载Windows x86-64 executable installer
  2. 下载完成后，双击python-x.x.x-amd64.exe,并且勾选”Add Python x.x to PATH”，并点击”Install Now”
  3. 当你看到下面这个对话框，则代表安装成功

安装 pip

  1. 下载get-pip.py (https://bootstrap.pypa.io/get-pip.py)
  2. 运行python get-pip.py

安装 virtualenvwrapper-win

  1. pip install virtualenvwrapper-win
  2. 更多关于virtualenvwrapper-win的信息，详见https://github.com/davidmarble/virtualenvwrapper-win/

安装 PyCharm

  1. 访问https://www.jetbrains.com/pycharm/download/ 下载相应的PyCharm版本
  2. 安装PyCharm，并接受默认选项

设置 PyCharm的字体大小

  1. File->Settings->Apperance
  2. 选中”Override default fonts by(not recommended)”，设置合适的字体大小
  3. 在Settings对话框中选择Editor->General，勾选
    • “Change font size with Ctrol + Mouse Wheel”
    • “Highlight current scope”
  4. 在Settings对话框中选择Editor->Apperance，勾选
    • “Show line numbers”
    • “Show method separators”
    • “Show white spaces”
  5. 在Editor->Colors & Fonts->Font中
    • 在Theme下拉框中选择”Twilight”并点击”Save As…”按钮
    • 在字体下拉框中选择”Source Code Pro”，并设置合适的字体大小

在单元测试中进行依赖注入

在单元测试中，通常用一下方法进行依赖注入:

  1. 在构造函数传入一个接口实例，并把它保存在成员变量中

  3. 在调用被测试的方法之前用如下的方法之一接受一个接口的实例
    • 通过方法的参数传入一个接口的实例(paramter injection)
    • 通过一个工厂类(factory class)
    • 通过一个本地的工厂方法(factory method)
